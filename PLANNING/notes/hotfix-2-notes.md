## Decisions & deferrals

### T1 ‚Äî Cookie hardening
- `SESSION_COOKIE_SECURE` gated on `not DEV_MODE` so local dev (`http://127.0.0.1`) still receives cookies. Production must NOT set DEV_MODE ‚Üí SECURE remains True.
- `SESSION_COOKIE_SAMESITE = "Lax"` chosen over `"Strict"` because Strict breaks links from the Stripe success/cancel redirects (cross-site ‚Üí same-site navigation drops Strict cookies on first hit).
- `register.html minlength` raised 6 ‚Üí 8 to match server `_password_strength_error`. UX bug, but worth catching at HTML layer too.

### T2 ‚Äî CSRF
- Used Flask-WTF's `CSRFProtect`. Chosen over rolling our own ‚Äî Flask-WTF is the de-facto standard and handles AJAX header lookups (`X-CSRFToken` / `X-CSRF-Token`) out of the box.
- `/webhook/stripe` explicitly exempted via `@csrf.exempt`. Stripe signs the body with HMAC-SHA256; CSRF protection on a webhook would only break the integration.
- Inline `<meta name="csrf-token" content="{{ csrf_token() }}">` added to a shared `<head>` block so AJAX callers can read it. Updated `index.html`'s JS where it POSTs to `/generate`.
- Added tokens to **all 16 POST forms** across templates.

### T3 ‚Äî Defensive bounds
- `_ensure_table_columns` now asserts the table name is in `{"users", "quotes"}`. Defense-in-depth ‚Äî not user-reachable, but future maintainers can't accidentally wire SQLi into the helper.
- Password cap at 128 chars. Reasonable upper bound; werkzeug pbkdf2 (600k iters) takes ~200ms on a 128-byte password, ~3s on a 4KB password. The cap closes the DoS.
- `_user_pdf_dir` now `mode=0o700` on makedirs + an explicit `os.chmod(d, 0o700)` so existing dirs (from before this change) get tightened on first touch. 0o700 is owner-only; the web user is the only legitimate reader anyway (PDFs flow through `/download`, not direct file serving).

### T4 ‚Äî Rate limiting + headers
- Flask-Limiter with `default_limits=[]` (no global limit; only explicit decorators). Storage: `memory://` for now ‚Äî single-process is fine. Sprint 5 candidate: Redis backend for multi-worker prod.
- Rate keys default to `get_remote_address`. Important: behind a reverse proxy this needs `ProxyFix` middleware so `X-Forwarded-For` is honored. Added a comment in `app.py`; actual `ProxyFix` install is part of Sprint 5 ops config.
- Flask-Talisman CSP:
  - `script-src`: self, `cdn.tailwindcss.com`, `js.stripe.com`. NO `'unsafe-inline'`.
  - `style-src`: self, `'unsafe-inline'` (template `<style>` blocks), `fonts.googleapis.com`.
  - `font-src`: self, `fonts.gstatic.com`.
  - `frame-src`: `js.stripe.com` (Stripe checkout iframe).
  - `connect-src`: self, `api.stripe.com`.
- `force_https` gated on `not DEV_MODE` ‚Äî same gate as cookie SECURE so the two stay coherent.

### T5 ‚Äî Deps + deploy notes
- `requirements.txt` generated by hand from the actual import surface, not `pip freeze`. Keeps dev-only deps (`locust`, `pytest`) out of the prod manifest.
- Versions pinned to current installed versions (Flask 3.1.3, Werkzeug 3.1.8, Jinja2 3.1.6, SQLAlchemy 2.0.49, stripe 15.1.0, fpdf2 2.8.7). Tightening to `~=` semver, not strict `==`, so security patches can flow.
